## Directory Purposes

**src/ - Application Source**
- Purpose: All TypeScript/TSX source code
- Contains: Components, state, engines, services, types, data
- Key files: `App.tsx` (root), `main.tsx` (entry), `index.css` (global styles)

**src/data/ - Game Content Definitions**
- Purpose: All game content as static data structures
- Contains: Crops, machines, skills, workers, chapters, recipes, regions
- Key files: `crops.ts`, `machines.ts`, `skills.ts`, `world.ts`, `chapters.ts`
- Auto-generated: Files created by `economy/generate_*.ts` scripts
- Manual: `narrative.ts` (story text), `villageFolk.ts` (worker definitions)

**src/engine/ - Game Mechanics**
- Purpose: Pure calculation functions and game loop
- Contains: Crop growth, machine production, production multipliers, tick cycle
- Key files:
  - `TickEngine.tsx`: React component running 200ms loop
  - `mechanics.ts`: Core calculations (calculatePlotGrowth, calculateMachineProduction, fmt)
  - `productionEngine.ts`: Production multiplier with 3 tunable knobs
  - `liveOps.ts`: Dynamic events and merchant system

**src/services/ - Business Logic & Integration**
- Purpose: Services that don't fit state or UI layers
- Contains: Event bus, external integrations, validation, market system
- Key files:
  - `eventBus.ts`: Pub/sub event system + React hooks
  - `appwrite.ts`: Appwrite backend client
  - `marketService.ts`: Dynamic pricing system
  - `unlockService.ts`: Unlock condition evaluation
  - `marketService.ts`: Price volatility and merchant offers

**src/store/ - State Management**
- Purpose: Zustand store with persistence
- Main file: `gameStore.ts` (root store, tick function)
- Location: `src/store/gameStore.ts`
- Combines 8 slices:
  - `slices/resourceSlice.ts`: Coins, gems
  - `slices/farmSlice.ts`: Plots, crops
  - `slices/automationSlice.ts`: Machines, workers, upgrades
  - `slices/metaSlice.ts`: Unlocked content, panels, prestige
  - `slices/storySlice.ts`: Chapters, bosses, quests, weapons
  - `slices/workerAssignmentSlice.ts`: Worker deployment
  - `slices/craftingSlice.ts`: Recipe crafting queue
  - `slices/trackingSlice.ts`: Event tracking for unlock conditions

**src/types/ - Type Definitions**
- Purpose: Shared TypeScript interfaces
- Contains: Game object types, slice interfaces, event types
- Key files:
  - `base.ts`: Plot, PlayerMachine
  - `game.ts`: GameState, slice interfaces
  - `unlock.ts`: UnlockCondition, UnlockRequirement
  - `worker.ts`: Worker bonuses
  - `regionChapter.ts`: Story types

**src/ui/ - React Components**
- Purpose: All user-facing React components
- Contains: Panel components, layout, sidebar, stats bar
- Key files:
  - `Layout.tsx`: Main layout wrapper
  - `PanelManager.tsx`: Routes active panel selection to component
  - `Sidebar.tsx`: Navigation sidebar
  - `StatsBar.tsx`: Top bar with coins/gems/progress
  - `VillageNews.tsx`: Event feed display
  - Panel components: `FarmPanel.tsx`, `MachinePanel.tsx`, `WorkerPanel.tsx`, `SkillTreePanel.tsx`, `RegionPanel.tsx`, `UpgradePanel.tsx`, `PrestigePanel.tsx`, `StoryBookPanel.tsx`, `MarketPanel.tsx`, `AchievementsPanel.tsx`

**src/assets/ - Static Media**
- Purpose: Images, icons, sprites (if any)
- Contains: Emoji references are mostly inline, but directory exists for images

**src/i18n/ - Internationalization**
- Purpose: Translation and localization
- Key file: `t.ts` (translation function)

**public/ - Static Files**
- Purpose: Served as-is by Vite dev server and included in builds
- Contains: favicon.ico, manifest.json, etc.

**scripts/ - Build & Utility Scripts**
- Purpose: Node scripts for orchestration
- Key file: `orchestrator.ts` (GSD build pipeline)
- Run: `npm run build:pipeline`

**simulation/ - Simulation & QA**
- Purpose: Testing and validation scripts
- Key files:
  - `qa_runner.ts`: Regression testing
  - `progression_sim.ts`: Progression path testing
  - `balance_sim.ts`: Economy balance validation
  - `machine_roi_sim.ts`: ROI calculations
- Run: `npm run simulate:qa`, `npm run simulate:progression`, etc.

**economy/ - Economy Generation**
- Purpose: Auto-generate game content from parameters
- Scripts: `generate_crops.ts`, `generate_machines.ts`, etc.
- Output: Creates files in `src/data/`
- Usage: Edit parameters, re-run to regenerate data with validations

## Naming Conventions

**Files:**
- React components: PascalCase with `.tsx` (e.g., `FarmPanel.tsx`)
- Services/utilities: camelCase with `.ts` (e.g., `eventBus.ts`, `unlockService.ts`)
- Data/config files: snake_case or camelCase with `.ts` (e.g., `crops.ts`, `machine_upgrades.ts`)
- Slices: Named with `Slice` suffix (e.g., `farmSlice.ts`)

**Directories:**
- lowercase (e.g., `src/`, `services/`, `ui/`, `data/`, `store/`, `types/`)

**React Components:**
- PascalCase function names (e.g., `FarmPanel`, `StatsBar`)
- Export as default: `export default FarmPanel`
- Type-safe props: `interface ComponentProps { ... }` or `React.FC<Props>`

**Functions:**
- camelCase (e.g., `calculatePlotGrowth`, `getSkillMultiplier`, `fmt`)
- Pure functions preferred (no side effects)
- Use prefix pattern: `calculate*`, `get*`, `is*`, `can*`

**Variables:**
- camelCase for all locals (e.g., `plotId`, `coinGain`, `isReady`)
- UPPER_SNAKE_CASE for constants (e.g., `DEFAULT_TUNABLES`, `SKILL_CAP`)

**Types:**
- PascalCase for interfaces and types (e.g., `GameState`, `Plot`, `PlayerMachine`)
- Suffix pattern: `*Slice` for store slices, `*Props` for component props

**Events:**
- UPPER_SNAKE_CASE (e.g., `COINS_CHANGED`, `CROP_PLANTED`, `BOSS_DEFEATED`)

## Where to Add New Code

**New Feature:**
- Primary code: `src/store/slices/` if state-based, or add method to existing slice
- Engine logic: `src/engine/mechanics.ts` or new file in `src/engine/`
- UI: New component in `src/ui/`, added to `PanelManager.tsx` if top-level panel
- Data: New arrays in `src/data/` (can be auto-generated by `economy/` scripts)
- Types: Add interface to `src/types/game.ts` or relevant file in `src/types/`

**New Component/Module:**
- Implementation: Create file in `src/ui/` (components) or `src/services/` (non-UI logic)
- Naming: PascalCase for components (`MyPanel.tsx`), camelCase for services (`myService.ts`)
- Example: Adding a new panel would be:
  1. Create `src/ui/MyPanel.tsx`
  2. Add case in `src/ui/PanelManager.tsx` switch statement
  3. Add button in `src/ui/Sidebar.tsx` to navigate to it

**Utilities:**
- Shared helpers: `src/engine/` (game-specific) or new service in `src/services/`
- Shared types: `src/types/`
- Format examples: `src/engine/mechanics.ts` exports `fmt()` for number formatting

**New Event Type:**
- Add to `GameEventType` union in `src/services/eventBus.ts`
- Add payload interface to `EventPayloads` in same file
- Emit with: `eventBus.emit('EVENT_NAME', { payload })`
- Subscribe with: `useGameEvent('EVENT_NAME', (event) => { ... })`

**New Store Slice:**
- Create file: `src/store/slices/mySlice.ts`
- Pattern:
  ```typescript
  export interface MySlice {
    myState: any;
    myAction: (param: type) => void;
  }
  export const createMySlice: StateCreator<GameState, [], [], MySlice> = (set, get) => ({
    myState: initialValue,
    myAction: (param) => set({ ... })
  });
  ```
- Add to `gameStore.ts`: import and include in store creator

**Game Content:**
- Auto-generate: Edit `economy/generate_crops.ts` (or machines, etc.), run script
- Manual edit: `src/data/narrative.ts` (story text), `villageFolk.ts` (worker defs)
- Format: Standard interface + array of objects with consistent schema

## Special Directories

**node_modules/ - Dependencies**
- Purpose: Installed packages from npm
- Generated: `npm install` creates this
- Committed: No (.gitignored)
- Edit: Never manually edit, use `npm install` or `package.json`

**dist/ - Build Output**
- Purpose: Production build output
- Generated: `npm run build` creates this
- Committed: No (.gitignored)
- Use: Deploy this folder to web hosting

**.planning/ - GSD Documentation**
- Purpose: Analysis documents for code guidance
- Contents: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, etc.
- Committed: Yes
- Auto-generated: By GSD map-codebase commands

**simulation/ - Testing Scripts**
- Purpose: Automated testing and balance validation
- Committed: Yes (but depends on dev dependencies)
- Run: `npm run simulate:qa`, `npm run simulate:balance`, etc.

**.claude/ - Agent Memory**
- Purpose: Claude agent context/memory files
- Generated: By Claude agents
- Committed: No (.gitignored)

**.github/workflows/ - CI/CD**
- Purpose: GitHub Actions automation
- Committed: Yes
- Edit: Add new workflows here for CI/CD
